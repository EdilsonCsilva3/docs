1. Pré-requisitos

  • Ubuntu 20.04 / 22.04 / 24.04 (ou similar)
  • Conta com permissão sudo
  • Conexão com internet para baixar pacotes

Abra um terminal (Ctrl+Alt+T) antes de executar os comandos abaixo.

↑ topo

2. Instalar Docker Engine (passo a passo)

Remover versões antigas (se houver): 
sudo apt remove docker docker-engine docker.io containerd runc -y
Atualizar índice de pacotes e instalar dependências: 
sudo apt update
sudo apt install ca-certificates curl gnupg lsb-release -y
Adicionar a chave GPG oficial do Docker: 
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
Adicionar o repositório oficial do Docker: 
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
Instalar Docker Engine, CLI e containerd: 
sudo apt update
sudo apt install docker-ce docker-ce-cli containerd.io -y

Se tudo correr bem, o serviço Docker será instalado. Para gerenciar o serviço:

sudo systemctl enable --now docker
sudo systemctl status docker

3. Instalar Docker Compose — duas opções

Opção A — Plugin oficial (recomendado nos sistemas recentes)

A partir de versões recentes do Docker, o Docker Compose está disponível como plugin: docker compose (observe o espaço).

sudo apt update
sudo apt install docker-compose-plugin -y

Agora você usa o compose como:

docker compose up -d
docker compose ps

Opção B — binário docker-compose (compatibilidade com scripts antigos)

Instala a versão standalone (v2+) — substitua a URL por uma versão específica se quiser.

sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
docker-compose --version

Se optar pelo binário, você chamará docker-compose (com hífen). Se instalou o plugin, use docker compose.

4. Testar a instalação

Execute o famoso container "hello-world" para verificar:

sudo docker run --rm hello-world

Se receber uma mensagem de boas-vindas do Docker, a instalação está correta.

Executar sem sudo (opcional)

Para evitar digitar sudo em cada comando, adicione seu usuário ao grupo docker:

sudo usermod -aG docker $USER
# Após este comando, faça logout/login ou reinicie a sessão

5. Extras úteis

  • Ver versão do Docker: 
    docker --version
  • Ver versão do Compose (plugin): 
    docker compose version
  • Limpar imagens/parados: 
    docker system prune -a
  • Logs de containers: 
    docker logs -f <container>

Se tiver dúvidas sobre permissões ou erros, cole o erro para diagnóstico.
